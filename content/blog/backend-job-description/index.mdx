انتخاب نامزد مناسب برای نقش **Backend Engineer** مستلزم شناسایی افرادی است که مهارت‌های فنی و نرم لازم را داشته باشند و با ارزش‌های سازمان هم‌راستا باشند. با حجم بالای درخواست‌ها، اولویت‌بندی در ارزیابی نامزدها ضروری است. در این راهنما خلاصه‌ای از مهارت‌ها و کیفیت‌هایی که به‌عنوان Hiring Manager برای یک Backend Engineer در نظر می‌گیریم، آورده شده است.

## خلاصه Job Description توسعه‌دهنده بک‌اند

- طراحی، توسعه و نگهداری سیستم‌های بک‌اند با Performance بالا، قابل اطمینان، مقیاس‌پذیر و امن و منطق سمت سرور.
- نظارت بر پروژه‌ها از طراحی تا استقرار و تحویل یک تجربه کاربری یکپارچه.
- نگهداری و بهینه‌سازی زیرساخت بک‌اند متناسب با نیازهای در حال تغییر پروژه.
- همکاری با Frontend Developers برای یکپارچه‌سازی المان‌های کاربری با منطق سمت سرور.
- نوشتن کد بک‌اند تمیز، استاندارد، قابل نگهداری، قابل تست و قابل استفاده مجدد برای حل مسائل کسب‌وکار و فنی.

- مشارکت در تیم‌های چابک و همکاری با Product Owners و توسعه‌دهندگان فرانت و دیتا.
- ساخت منطق سمت سرور اپلیکیشن وب.

**الزامات:**

- تجربه جدی بک‌اند با الگوهای معماری Microservice و Event-Driven.
- مدرک علوم کامپیوتر یا رشته مرتبط.
- تجربه حرفه‌ای با حداقل یک زبان بک‌اند (مثلاً Python، C#، Java).
- تجربه کار با Docker و فناوری‌های Container مثل Kubernetes.
- درک عمیق از Continuous Integration و Continuous Delivery.
- تجربه عملی با مفاهیم پیشرفته طراحی مثل DDD و OOP.
- مهارت ارتباط و همکاری قوی.

## مسئولیت‌های کلیدی Backend Developer

- **Server-side development:** نوشتن منطق اپلیکیشن سمت سرور برای هندل کردن درخواست‌ها، ارتباط با دیتابیس و منطق کسب‌وکار.
- **API development:** طراحی، پیاده‌سازی و نگهداری API برای ارتباط بین سرویس‌ها و سیستم‌های خارجی. ابزارهایی مثل Swagger برای طراحی و مستندسازی API استفاده می‌شوند.
- **Database administration and management:** انتخاب نوع دیتابیس (SQL یا NoSQL)، انتخاب ORM مناسب، نوشتن کوئری‌های بهینه و اطمینان از امنیت داده.
- **Performance optimization:** شناسایی و رفع گلوگاه‌ها (کوئری‌های کند، کیفیت کد، Latency بالا).
- **Integration of third-party services:** یکپارچه‌سازی درگاه پرداخت، سرویس‌های Cloud و سرویس‌های احراز هویت مثل Auth0.
- **Testing and bug fixing:** نوشتن Unit، Integration و E2E Test و رفع باگ‌ها.
- **Documentation:** نگارش و به‌روزرسانی مستندات فنی و API.
- **Collaboration:** همکاری با Frontend، طراحان و مدیران محصول.

## مهارت‌های فنی و نرم مورد انتظار

- تسلط به حداقل یک زبان برنامه‌نویسی بک‌اند.
- درک عمیق از دیتابیس‌ها و Caching.
- دانش API و طراحی Endpoint.
- آشنایی با Web Serverها (مثل Nginx).
- آشنایی با Design Patternها.
- تسلط به Version Control (مثلاً Git و GitHub/GitLab).
- رعایت بهترین شیوه‌های امنیت (جلوگیری از SQL Injection، XSS، CSRF).
- مهارت ارتباط و کار تیمی.

## سوالات رایج مصاحبه Backend

- متدهای مختلف HTTP چیست؟ (GET، POST، PUT، DELETE و غیره.)
- CI/CD را چطور تعریف می‌کنید؟
- انواع دیتابیس‌ها و تفاوت SQL و NoSQL؟
- تفاوت برنامه‌نویسی Synchronous و Asynchronous؟
- نقش Caching در بک‌اند و چه استراتژی‌های Caching استفاده کرده‌اید؟
- آسیب‌پذیری‌های رایج امنیتی وب و راه‌های کاهش آن‌ها؟
